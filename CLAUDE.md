## What this repo is

A real-time multiplayer "Crash" casino game built as a Bun monorepo: two NestJS (Fastify) backends behind Kong, a TanStack Start frontend, Postgres + RabbitMQ + Logto + Prometheus/Grafana on the side. See [`README.md`](./README.md) for the full overview, quick start, and demo credentials.

## Workspace layout


## Commands

Root-level only. Per-workspace scripts (`dev`, `test`, `migration:*`, etc.) live in each workspace's `package.json` and `CLAUDE.md`.

```bash
bun run docker:up        # full stack + Logto seed + frontend build
bun run docker:up:infra  # everything except the frontend container
bun run docker:down      # stop containers, keep volumes
bun run docker:prune     # nuke containers, volumes, images
bun run seed:logto       # re-run Logto seeding against a running stack
bun run lint             # biome check .
bun run format           # biome format --write .
bun run typecheck        # tsc across both services
bun run test:unit        # vitest across both services
```

## Tooling conventions

- **Bun** is the runtime and package manager. Do not invoke `npm`, `pnpm`, or `yarn`. Any stray `pnpm` keys in `package.json` files are vestigial.
- **Biome 2.4** for lint + format: tabs, double quotes, organize-imports on. Run `bun run lint` before declaring work done.
- **TypeScript strict** everywhere. `any` requires justification — prefer `unknown` + narrowing, branded types, or a proper generic.
- **MikroORM v7** in both services. v7 dropped class decorators: define entities with `defineEntity()` + property builders, and register the resulting **schema** (not the class) in `mikro-orm.config.ts`.

## Container runtime

This dev machine runs **Podman under WSL**. `scripts/compose.ts` auto-detects `docker` vs `podman` and shells out accordingly — keep new compose calls going through it.

Testcontainers does not pick the Podman socket up automatically. Integration suites must:

- Read an explicit `DOCKER_HOST=tcp://...` from env.
- `skipIf` the host is unreachable so CI and laptops without a running engine don't false-fail.

## Architecture pointers

One-liners — see [`docs/architecture.md`](./docs/architecture.md) and [`docs/RFC.md`](./docs/RFC.md) for the full picture.

- **Outbox / inbox pattern** between services: producers write to an outbox table in the same TX as the business state; a relay publishes to RabbitMQ; consumers dedupe via an inbox table.
- **Money is BIGINT cents** at the DB and DTO layer. `services/wallets` uses `big.js` for any arithmetic that crosses decimal boundaries — never `number`.
- **Multiplier is client-projected.** The server emits `round.started { startTime, k }`; the frontend computes `m(t) = e^(k * (now - startTime))` on `requestAnimationFrame`. The server never streams per-frame multiplier values.
- **Kong validates JWTs** against Logto's JWKS at the gateway. Services trust `x-userinfo`-style headers from Kong and re-verify the `aud` claim.
- **Transport split:** REST for queries and user-initiated commands (place bet, cash out); **RabbitMQ for cross-service events** (round lifecycle, debit/credit, payouts). Wallet debit/credit is never called over REST from `games` — always via the event bus.
- **Provably-fair** seed chain lives in `services/games`. The deterministic seed table is generated by `scripts/generate-seed-table.ts`; E2E scenarios live under `scripts/scenarios/`.

## Project decisions to not relitigate

- **Logto for OIDC.** Project uses Logto as the IdP. Don't propose swapping to Keycloak or another provider.
- **The user runs their own commits.** Do not invoke commit skills, do not run `git commit`/`git add` on your own — leave changes staged or unstaged for the user to review.
- **TDD for important use cases.** Write tests first for application-layer use cases, domain logic, money flows, and the provably-fair seed chain. Throwaway scripts and pure UI tweaks are exempt.
- **Prefer MikroORM QueryBuilder** (`em.createQueryBuilder(...)`) over raw SQL for joins and aggregates. Drop to raw SQL only when QB genuinely can't express the query.
